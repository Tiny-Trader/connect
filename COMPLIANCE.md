# License Compliance Guide (Apache-2.0)

This file is a practical checklist for teams using or modifying `tt-connect`.

See also: `DISCLAIMER.md` for financial-risk and liability notices.

## If You Redistribute the Software

1. Keep copyright and license notices.
2. Include a copy of the Apache License 2.0.
3. Document material changes made by your organization in modified files.
4. Preserve applicable third-party attribution notices.

## If You Run It as a Network Service

Apache-2.0 does not require you to publish source code merely because the software is used over a network.

Recommended minimum:

1. Track the exact running source version internally.
2. Preserve license and attribution notices in distributed artifacts.
3. Review trademark usage separately from copyright licensing.

## Internal Compliance Controls

1. Track upstream version and local patches.
2. Maintain SBOM/dependency inventory for releases.
3. Add release checklist item for Apache-2.0 notice preservation.
4. Keep legal review for licensing and trademark usage.

## Enforcement Workflow (Project Side)

1. Detect potential non-compliance.
2. Collect dated evidence.
3. Request correction of notice, attribution, or redistribution issues.
4. Escalate via legal channels if unresolved.

Note: This document is operational guidance, not legal advice.
